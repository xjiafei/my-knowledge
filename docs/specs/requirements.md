# 需求规格 — 个人知识库管理系统

## 1. 背景与目标

### 背景
个人用户需要一个集中管理知识笔记的 Web 系统，支持 Markdown 编写、标签分类和全文搜索，替代分散的本地文件管理方式。

### 目标
- 提供笔记全生命周期管理（创建、编辑、查看、删除）
- 支持 Markdown 实时编辑与预览
- 通过标签和分类组织知识体系
- 通过全文搜索快速定位笔记

### 成功指标
| 指标 | 目标值 |
|------|--------|
| 笔记创建到保存完成（POST /notes P95） | < 1 秒 |
| 全文搜索返回结果（P95） | < 1 秒 |
| 系统首屏加载 | < 2 秒 |
| 新用户完成首篇笔记创建 | < 5 分钟（无需培训）|

## 2. 用户角色

本系统为**单用户系统**（个人知识库），仅有一个角色：

| 角色 | 核心目标 | 使用场景 | 使用频率 |
|------|---------|---------|---------|
| 知识库所有者 | 记录、组织、检索个人知识 | 日常学习记录、工作笔记整理、技术资料归档、通过搜索快速回顾知识点 | 每天 |

### 权限矩阵

| 操作 | 所有者 |
|------|--------|
| 笔记 CRUD | Yes |
| 标签 CRUD | Yes |
| 分类 CRUD | Yes |
| 全文搜索 | Yes |

> 单用户系统无需登录认证（P0 阶段）。

## 3. 功能需求（用户故事）

### P0 — Must Have

#### US-001: 创建笔记
**作为**知识库所有者，**我想要**创建一篇新笔记，**以便**记录知识。

**验收标准：**
1. **Given** 用户在笔记列表页 **When** 点击"新建笔记" **Then** 进入编辑页，标题和内容为空
2. **Given** 用户在编辑页 **When** 填写标题和 Markdown 内容后点击"保存" **Then** 笔记保存成功，跳转到笔记详情页��显示成功提示
3. **Given** 用户在编辑页 **When** 标题为空时点击"保存" **Then** 显示"标题不能为空"错误提示，不保存

**优先级：** P0

---

#### US-002: 编辑笔记
**作为**知识库所有者，**我想要**编辑已有笔记，**以便**更新知识内容。

**验收标准：**
1. **Given** 用户在笔记详情页 **When** 点击"编辑" **Then** 进入编辑页，标题和内容回填当前值
2. **Given** 用户修改内容后 **When** 点击"保存" **Then** 笔记更新成功，显示成功提示
3. **Given** 用户修改内容后 **When** 点击"取消" **Then** 放弃修改，返回详情页

**优先级：** P0

---

#### US-003: 删除笔记
**作为**知识库所有者，**我想要**删除不需要的笔记，**以便**保持知识库整洁。

**验收标准：**
1. **Given** 用户在笔记列表或详情页 **When** 点击"删除" **Then** 弹出确认对话框
2. **Given** 确认对话框显示 **When** 点击"确认删除" **Then** 笔记被删除，列表刷新，显示成功提示
3. **Given** 确认对话框显示 **When** 点击"取消" **Then** 笔记不被删除，对话框关闭

**优先级：** P0

---

#### US-004: 查看笔记列表
**作为**知识库所有者，**我想要**查看所有笔记的列表，**以便**浏览和管理笔记。

**验收标准：**
1. **Given** 用户进入系统 **When** 访问首页 **Then** 显示笔记列表，每项显示标题、摘要（Markdown 原文前 100 个字符）、更新时间、标签
2. **Given** 笔记列表 **When** 笔记数量超过 20 条 **Then** 分页显示，每页 20 条
3. **Given** 笔记列表 **When** 无笔记 **Then** 显示空状态提示"暂无笔记，点击新建开始记录"

**优先级：** P0

---

#### US-005: Markdown 实时预览
**作为**知识库所有者，**我想要**在编辑 Markdown 时实时看到渲染效果，**以便**确认排版正确。

**验收标准：**
1. **Given** 用户在编辑页 **When** 输入 Markdown 文本 **Then** 右侧实时显示渲染后的 HTML 预览
2. **Given** 用户输入包含代码块、表格、列表的 Markdown **When** 查看预览 **Then** 代码块有语法高亮，表格和列表正确渲染
3. **Given** 用户在笔记详情页 **When** 查看笔记 **Then** 内容以渲染后的 HTML 展示（非原始 Markdown）

**优先级：** P0

---

#### US-006: 全文搜索
**作为**知识库所有者，**我想要**通过关键词搜索笔记，**以便**快速找到需要的知识。

**验收标准：**
1. **Given** 用户在任意页面 **When** 在搜索框输入关键词并提交 **Then** 显示标题或内容匹配的笔记列表，关键词高亮
2. **Given** 一篇笔记标题不含关键词但内容含关键词 **When** 搜索该关键词 **Then** 该笔记出现在结果列表中（内容全文搜索）
3. **Given** 搜索结果 **When** 无匹配笔记 **Then** 显示"未找到相关笔记"空状态提示
4. **Given** 搜索结果 **When** 点击某条结果 **Then** 跳转到该笔记详情页

**优先级：** P0

---

#### US-007: 标签管理
**作为**知识库所有者，**我想要**创建和管理标签，**以便**对笔记进行分类标记。

**验收标准：**
1. **Given** 用户在标签管理页 **When** 输入标签名称并提交 **Then** 标签创建成功，列表刷新
2. **Given** 用户在标签管理页 **When** 标签名称为空或与已有标签重复 **Then** 显示相应错误提示
3. **Given** 用户在编辑笔记时 **When** 选择或输入标签 **Then** 标签关联到该笔记，一篇笔记可关联多个标签（上限 20 个）
4. **Given** 用户在标签管理页 **When** 删除一个标签 **Then** 标签被删除，已关联该标签的笔记解除关联（笔记不删除）

**优先级：** P0

---

#### US-008: 按标签筛选笔记
**作为**知识库所有者，**我想要**按标签筛选笔记，**以便**查看某一主题的所有笔记。

**验收标准：**
1. **Given** 用户在笔记列表页 **When** 点击某个标签 **Then** 列表仅显示包含该标签的笔记
2. **Given** 标签筛选激活 **When** 该标签下无笔记 **Then** 显示"该标签下暂无笔记"
3. **Given** 标签筛选激活 **When** 点击"清除筛选" **Then** 恢复显示全部笔记

**优先级：** P0

---

### P1 — Should Have

#### US-009: 分类管理（层级）
**作为**知识库所有者，**我想要**创建层级分类（如"技术/前端"），**以便**用树形结构组织知识。

**验收标准：**
1. **Given** 分类管理页 **When** 创建分类时可选择父分类 **Then** 形成最多 3 级的树形结构
2. **Given** 编辑笔记时 **When** 选择分类 **Then** 笔记归属该分类，一篇笔记只属于一个分类
3. **Given** 用户点击删除含子分类的父分类，系统弹出对话框提示"将同时删除所有子分类，相关笔记分类将置空" **When** 用户点击"确认删除" **Then** 该分类及其所有子分类被删除，相关笔记的分类字段置空

**优先级：** P1

---

#### US-010: 笔记排序
**作为**知识库所有者，**我想要**按不同维度排序笔记列表，**以便**灵活浏览。

**验收标准：**
1. **Given** 笔记列表页 **When** 选择排序方式 **Then** 支持按更新时间（默认，降序）、创建时间（降序）、标题字母（升序 A-Z）排序，选择后列表立即刷新

**优先级：** P1

---

#### US-011: 标签云
**作为**知识库所有者，**我想要**在侧边栏看到标签云，**以便**直观了解知识分布。

**验收标准：**
1. **Given** 侧边栏 **When** 展示标签云 **Then** 标签按关联笔记数量调整字体大小，点击标签触发筛选

**优先级：** P1

---

**优先级：** P2（本期不实现）

---

#### US-013: 知识库统计
**作为**知识库所有者，**我想要**查看笔记数量、标签分布等统计，**以便**了解知识库概况。

**优先级：** P2（本期不实现）

## 4. 非功能需求

### 性能
| 指标 | 目标值 |
|------|--------|
| 首屏加载 | < 2 秒 |
| API 响应时间（P95） | < 500ms |
| 全文搜索响应（1000 篇笔记内） | < 1 秒 |
| 支持笔记总量 | >= 10,000 篇 |
| 并发用户数 | 1（单用户系统，无并发场景） |

### 安全
- 单用户系统，P0 阶段无登录认证
- API 不暴露系统内部信息（如堆栈跟踪）
- Markdown 渲染时过滤危险标签（XSS 防护）
- 使用参数化查询防止 SQL 注入

### 可用性
- 本地部署，可用性依赖宿主机
- 数据存储在 MySQL，支持 mysqldump 备份
- 前端支持：Chrome 90+、Firefox 90+、Edge 90+
- 最小分辨率：1280x720

### 可维护性
- 前后端分离，独立部署
- RESTful API 风格
- 代码有基本注释，关键逻辑有说明

## 5. 边界与排除项

### 本期包含
- 笔记 CRUD + Markdown 编辑预览
- 标签多对多关联 + 筛选 + 标签云
- 分类树（最多 3 级）
- 全文搜索（基于 MySQL FULLTEXT 或 LIKE）
- 单用户，无认证

### 本期不做
- 多用户 / 登录认证 / 权限管理
- 笔记协作 / 分享 / 公开发布
- 文件附件 / 图片上传
- 笔记版本历史 / 回滚
- 移动端适配
- 国际化（i18n）
- 笔记间双向链接（Wiki 链接）
- AI 辅助写作

## 6. 术语表

| 术语 | 定义 |
|------|------|
| 笔记（Note） | 系统核心实体，包含标题、Markdown 内容、所属分类、关联标签、创建时间、更新时间 |
| 标签（Tag） | 轻量分类标记，与笔记多对多关系，用于快速筛选 |
| 分类（Category） | 树形层级结构（最多 3 级），与笔记多对一关系，用于系统化组织 |
| Markdown | 轻量标记语言，用于编写笔记内容，在详情页和预览区渲染为 HTML |
| 全文搜索 | 在笔记标题和内容中匹配关键词并返回结果，支持关键词高亮 |
| 标签云（Tag Cloud） | 标签的可视化展示组件，标签字体大小与关联笔记数成正比 |
| 摘要（Summary） | 笔记 Markdown 原文的前 100 个字符（不含渲染后 HTML 标签），用于列表页预览 |

## 7. 前置假设

- 系统部署在本地或单台服务器，不考虑分布式场景
- 单用户使用，无并发写入冲突
- MySQL 版本 >= 5.7（支持 FULLTEXT 全文索引）
- 用户具备基本 Markdown 语法知识
