# 产品设计 — 个人知识库管理系统

## 1. 信息架构

### 导航结构

```
顶部导航栏（固定）
├── Logo / 系统名称 → 点击回到笔记列表页
├── 全局搜索框 → 输入关键词提交后跳转搜索结果页
└── "新建笔记"按钮 → 跳转笔记编辑页（新建模式）

页面布局：顶部导航 + 侧边栏 + 主内容区
├── 侧边栏（左侧，可折叠）
│   ├── 分类树（树形展开，点击筛选）
│   ├── 标签云（字体大小按笔记数量加权）
│   └── 快捷入口：标签管理、分类管理
└── 主内容区（右侧）
    ├── 笔记列表页（首页，默认路由 /）
    ├── 笔记详情页（/notes/:id）
    ├── 笔记编辑页（/notes/new 或 /notes/:id/edit）
    ├── 标签管理页（/tags）
    ├── 分类管理页（/categories）
    └── 搜索结果页（/search?q=关键词）
```

### 权限说明

本系统为单用户系统，无角色区分。所有页面和操作对唯一用户（知识库所有者）完全开放，无需鉴权。P0 阶段不设登录认证，系统启动后直接进入笔记列表页。

### 筛选规则说明

- 标签筛选和分类筛选可同时激活，关系为 **AND**（即同时满足所选标签和所选分类）
- 同时激活时，顶部显示两个筛选标识，"清除筛选"一键清除全部
- 搜索结果页不受侧边栏筛选影响，独立显示搜索结果

### 路由表

| 路由 | 页面 | 说明 |
|------|------|------|
| `/` | 笔记列表页 | 首页 |
| `/notes/new` | 笔记编辑页 | 新建模式 |
| `/notes/:id` | 笔记详情页 | 查看笔记 |
| `/notes/:id/edit` | 笔记编辑页 | 编辑模式 |
| `/tags` | 标签管理页 | 标签 CRUD |
| `/categories` | 分类管理页 | 分类 CRUD |
| `/search?q=xxx` | 搜索结果页 | 搜索结果 |

## 2. 页面清单

### 2.1 笔记列表页（首页）

**页面目的：** 浏览全部笔记，支持标签筛选、分类筛选、排序和分页。

**布局：** 侧边栏（分类树 + 标签云） + 主内容区（工具栏 + 笔记卡片列表 + 分页器）

**核心操作：**
- 浏览笔记列表，点击标题进入详情页
- 点击侧边栏标签/分类筛选笔记
- 切换排序方式（更新时间/创建时间/标题字母）
- 点击卡片右侧"删除"图标按钮删除笔记（弹出确认框）；点击"编辑"图标进入编辑页
- 点击"新建笔记"进入编辑页

**数据展示：** 每条笔记卡片显示：标题、摘要（Markdown 原文前 100 个字符）、更新时间、关联标签列表。默认按更新时间降序，每页 20 条。

**交互流程：**
1. 进入页面 → 加载笔记列表（默认全部、按更新时间降序）
2. 点击侧边栏标签 → 列表刷新为该标签下的笔记，顶部显示筛选标识和"清除筛选"按钮
3. 点击分类树节点 → 列表刷新为该分类下的笔记
4. 切换排序下拉框 → 列表按选中规则重新排序
5. 点击笔记卡片"删除" → 弹出确认框 → 确认后删除并刷新列表

**异常场景：**
- 无笔记：显示空状态插图 + 文案"暂无笔记，点击新建开始记录" + 新建按钮
- 标签筛选无结果：显示"该标签下暂无笔记"
- 删除失败：Toast 提示"删除失败，请重试"
- 网络异常：Toast 提示"网络异常，请检查连接"

**用户故事追溯：** US-001, US-003, US-004, US-006, US-008, US-010, US-011

---

### 2.2 笔记详情页

**页面目的：** 查看笔记完整内容（Markdown 渲染后的 HTML）。

**布局：** 顶部操作栏（编辑、删除、返回列表按钮） + 主内容区（标题 + 元信息 + 渲染内容）

**核心操作：**
- 查看笔记渲染后内容（代码块语法高亮、表格、列表正确渲染）
- 点击"编辑"进入编辑页
- 点击"删除"删除笔记
- 点击标签跳转到列表页并触发该标签筛选

**数据展示：** 标题、所属分类、关联标签、创建时间、更新时间、Markdown 渲染后的 HTML 正文。

**交互流程：**
1. 从列表页/搜索结果页点击笔记 → 进入详情页，加载笔记数据
2. 点击"编辑" → 跳转 `/notes/:id/edit`
3. 点击"删除" → 确认框 → 确认后删除并跳转列表页

**异常场景：**
- 笔记不存在（ID 无效）：显示 404 提示 + 返回列表按钮
- 删除失败：Toast 提示"删除失败，请重试"

**用户故事追溯：** US-002, US-003, US-005

---

### 2.3 笔记编辑页（新建/编辑合并）

**页面目的：** 创建或编辑笔记，支持 Markdown 实时预览。

**布局：** 顶部（标题输入框 + 分类选择 + 标签选择） + 下方左右分栏（左侧 Markdown 编辑器 + 右侧实时预览） + 底部操作栏（保存、取消）

**核心操作：**
- 输入标题（必填）
- 编写 Markdown 内容，右侧实时渲染预览
- 选择分类（下拉树，可选）
- 选择/输入标签（多选，上限 20 个）
- 保存或取消

**数据展示：**
- 新建模式：标题和内容为空，分类和标签为空
- 编辑模式：回填当前笔记的标题、内容、分类、标签

**交互流程：**
1. 进入页面 → 新建模式加载空表单 / 编辑模式加载已有数据
2. 输入 Markdown → 右侧预览区实时渲染（代码块语法高亮、表格、列表）
3. 选择标签 → 从已有标签下拉列表选择（不支持在编辑页直接创建新标签，如需新建请先去标签管理页创建）
4. 点击"保存" → 校验标题非空 → 调用 API → 成功后跳转详情页 + Toast 提示"保存成功"
5. 点击"取消" → 新建模式返回列表页 / 编辑模式返回详情页

**异常场景：**
- 标题为空点击保存：标题输入框下方显示"标题不能为空"红色提示
- 标签超过 20 个：提示"最多关联 20 个标签"
- 保存失败：Toast 提示"保存失败，请重试"，不跳转
- 笔记不存在（编辑模式 ID 无效）：显示 404 提示 + 返回列表按钮

**用户故事追溯：** US-001, US-002, US-005, US-007

---

### 2.4 标签管理页

**页面目的：** 集中管理所有标签，支持新建、重命名、删除。

**布局：** 顶部新建区域（输入框 + 添加按钮） + 标签列表（表格：标签名、关联笔记数、操作列）

**核心操作：**
- 创建新标签（输入名称，提交）
- 重命名标签（行内编辑）
- 删除标签（确认后删除，解除与笔记的关联）

**数据展示：** 标签名称、关联笔记数量。按关联笔记数降序排列。

**交互流程：**
1. 输入标签名 → 点击"添加" → 创建成功，列表刷新
2. 点击标签行的"编辑"图标 → 行内输入框激活，修改名称 → 回车或点击"确认"保存；按 Esc 或点击"取消"放弃
3. 点击某标签的"删除" → 确认框提示"删除后相关笔记将解除该标签" → 确认后删除

**异常场景：**
- 标签名为空：提示"标签名称不能为空"
- 标签名重复：提示"该标签已存在"
- 无标签：显示"暂无标签，请创建第一个标签"
- 删除失败：Toast 提示"删除失败，请重试"

**用户故事追溯：** US-007

---

### 2.5 分类管理页

**页面目的：** 管理层级分类（最多 3 级），支持新建、编辑、删除。

**布局：** 左侧分类树（可展开折叠） + 右侧操作区（新建分类表单：名称 + 父分类下拉）

**核心操作：**
- 创建分类（输入名称，可选父分类，最多 3 级）
- 编辑分类名称
- 删除分类（含子分类级联删除，相关笔记分类置空）

**数据展示：** 树形展示所有分类，每个节点显示名称和关联笔记数。

**交互流程：**
1. 输入分类名 + 选择父分类（可选） → 点击"创建" → 树刷新
2. 点击节点"删除" → 若有子分类，确认框提示"将同时删除所有子分类，相关笔记分类将置空" → 确认后级联删除
3. 选择父分类时，若已有 3 级则第 3 级节点不可选为父分类

**异常场景：**
- 分类名为空：提示"分类名称不能为空"
- 超过 3 级：父分类下拉中不显示第 3 级节点
- 无分类：显示"暂无分类，请创建第一个分类"
- 删除失败：Toast 提示"删除失败，请重试"

**用户故事追溯：** US-009

---

### 2.6 搜索结果页

**页面目的：** 展示全文搜索结果，关键词高亮显示。

**布局：** 顶部搜索框（回填关键词） + 结果统计 + 结果列表（与笔记列表页卡片格式一致，关键词高亮）

**核心操作：**
- 查看搜索结果，点击进入笔记详情页
- 修改关键词重新搜索

**数据展示：** 匹配笔记的标题（关键词高亮）、内容摘要（关键词高亮）、更新时间、标签。显示"共找到 N 条结果"。

**交互流程：**
1. 在任意页面搜索框输入关键词回车 → 跳转搜索结果页，显示匹配结果
2. 点击某条结果 → 跳转笔记详情页
3. 修改搜索框关键词回车 → 刷新结果

**异常场景：**
- 无匹配结果：显示"未找到相关笔记" + 建议文案"试试其他关键词"
- 关键词为空提交：不跳转，搜索框提示"请输入搜索关键词"
- 关键词超过 200 字符：截断至 200 字符后提交搜索
- 搜索失败：Toast 提示"搜索失败，请重试"

**用户故事追溯：** US-006

## 3. 核心用户旅程

### 3.1 首次使用：新建笔记

```
笔记列表页（空状态）
  → 点击"新建笔记"按钮
  → 笔记编辑页（新建模式，空表单）
  → 输入标题，编写 Markdown 内容，右侧实时预览
  → 可选：选择标签、分类
  → 点击"保存"
  → 笔记详情页（查看渲染后内容，Toast "保存成功"）
```

### 3.2 标签筛选

```
笔记列表页
  → 点击侧边栏标签云中某标签（如"Vue"）
  → 列表刷新，仅显示含"Vue"标签的笔记，顶部显示筛选标识
  → 点击某笔记标题
  → 笔记详情页
  → 点击"返回"或浏览器后退
  → 回到筛选后的列表页
  → 点击"清除筛选"恢复全部
```

### 3.3 全文搜索

```
任意页面（顶部导航栏搜索框始终可用）
  → 输入关键词"Spring Boot"，回车
  → 搜索结果页，显示匹配笔记列表，关键词高亮
  → 点击某条结果
  → 笔记详情页（查看完整内容）
```

## 4. API 概要清单

### 笔记

| 方法 | 路径 | 说明 | 对应故事 |
|------|------|------|---------|
| GET | `/api/notes` | 笔记列表（支持分页、排序、标签筛选、分类筛选） | US-004, US-008, US-010 |
| POST | `/api/notes` | 创建笔记 | US-001 |
| GET | `/api/notes/:id` | 笔记详情 | US-005 |
| PUT | `/api/notes/:id` | 更新笔记 | US-002 |
| DELETE | `/api/notes/:id` | 删除笔记 | US-003 |
| GET | `/api/notes/search?q=keyword` | 全文搜索 | US-006 |

**GET /api/notes 查询参数：**
- `page`（页码，默认 1）、`size`（每页条数，默认 20）
- `sort`（排序字段：`updatedAt`/`createdAt`/`title`，默认 `updatedAt`）
- `order`（排序方向：`desc`/`asc`，默认 `desc`；`title` 默认 `asc`）
- `tagId`（按标签 ID 筛选）
- `categoryId`（按分类 ID 筛选）

### 标签

| 方法 | 路径 | 说明 | 对应故事 |
|------|------|------|---------|
| GET | `/api/tags` | 标签列表（含关联笔记数） | US-007, US-011 |
| POST | `/api/tags` | 创建标签 | US-007 |
| PUT | `/api/tags/:id` | 更新标签 | US-007 |
| DELETE | `/api/tags/:id` | 删除标签 | US-007 |

### 分类

| 方法 | 路径 | 说明 | 对应故事 |
|------|------|------|---------|
| GET | `/api/categories` | 分类树（含关联笔记数） | US-009 |
| POST | `/api/categories` | 创建分类 | US-009 |
| PUT | `/api/categories/:id` | 更新分类 | US-009 |
| DELETE | `/api/categories/:id` | 删除分类（级联删除子分类） | US-009 |

### 通用响应格式

```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```

错误响应：

```json
{
  "code": 400,
  "message": "标题不能为空",
  "data": null
}
```

## 5. 数据模型概要

### Note（笔记）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | BIGINT | PK, AUTO_INCREMENT | 主键 |
| title | VARCHAR(200) | NOT NULL | 标题 |
| content | LONGTEXT | | Markdown 原文 |
| category_id | BIGINT | FK, NULLABLE | 所属分类 |
| created_at | DATETIME | NOT NULL | 创建时间 |
| updated_at | DATETIME | NOT NULL | 更新时间 |

### Tag（标签）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | BIGINT | PK, AUTO_INCREMENT | 主键 |
| name | VARCHAR(50) | NOT NULL, UNIQUE | 标签名称 |
| created_at | DATETIME | NOT NULL | 创建时间 |

### Category（分类）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | BIGINT | PK, AUTO_INCREMENT | 主键 |
| name | VARCHAR(100) | NOT NULL | 分类名称 |
| parent_id | BIGINT | FK, NULLABLE | 父分类 ID（NULL 为顶级） |
| created_at | DATETIME | NOT NULL | 创建时间 |

### note_tags（笔记-标签关联表）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| note_id | BIGINT | FK, NOT NULL | 笔记 ID |
| tag_id | BIGINT | FK, NOT NULL | 标签 ID |

联合主键：(note_id, tag_id)

### 索引策略

- `note` 表：`content` 字段建 FULLTEXT 索引用于全文搜索；`category_id` 建普通索引；`updated_at` 建索引用于排序
- `category` 表：`parent_id` 建索引用于树查询
- `note_tags` 表：`tag_id` 建索引用于按标签查笔记

### ER 关系

```
Note *---* Tag        （多对多，通过 note_tags）
Note *---1 Category   （多对一，一篇笔记属于一个分类）
Category *---1 Category （自引用，parent_id 实现层级）
```
