# 需求规格 — 文件类型知识（增量特性）

## 1. 背景与目标

### 背景
现有个人知识库系统仅支持 Markdown 笔记。用户在日常学习和工作中大量使用 PDF、Office 文档、图片等文件类型，需要将这些文件纳入知识库统一管理，而不是散落在本地文件系统的不同位置。

### 目标
- 支持将常见文件（PDF、Office、图片等）上传到知识库，作为一类知识进行管理
- 提供文件查看（元数据）、下载和删除功能
- 与现有标签体系集成，支持对文件打标签、筛选

### 成功指标
| 指标 | 目标值 |
|------|--------|
| 单文件上传（≤ 10MB）完成时间（P95） | < 3 秒 |
| 文件列表首屏加载（P95） | < 1 秒 |
| 文件下载响应（启动下载，P95） | < 1 秒 |
| 用户完成首次文件上传 | < 3 分钟（无需培训） |

## 2. 用户角色

与现有系统一致，本系统为**单用户系统**，角色为**知识库所有者**。

### 权限矩阵（增量）

| 操作 | 所有者 |
|------|--------|
| 文件上传 | Yes |
| 文件列表查看 | Yes |
| 文件下载 | Yes |
| 文件删除 | Yes |
| 文件打标签 | Yes |

## 3. 功能需求（用户故事）

### P0 — Must Have

#### US-F001: 上传文件
**作为**知识库所有者，**我想要**上传文件到知识库，**以便**将文件类型的知识纳入统一管理。

**验收标准：**
1. **Given** 用户在文件管理页 **When** 点击"上传文件"并选择文件 **Then** 文件上传成功，文件列表刷新，显示成功提示
2. **Given** 用户选择文件上传 **When** 文件格式不在支持列表中 **Then** 上传被拒绝，显示"不支持该文件格式，支持：PDF、Word、Excel、PPT、图片（JPG/PNG/GIF/WEBP）、纯文本、Markdown、ZIP"
3. **Given** 用户选择文件上传 **When** 文件大小超过 100MB **Then** 上传被拒绝，显示"文件大小不能超过 100MB"
4. **Given** 用户上传文件 **When** 可选填写文件描述（最多 500 字符）**Then** 描述与文件一起保存，在文件列表中展示
5. **Given** 用户上传文件 **When** 同名文件已存在 **Then** 允许上传（服务端使用 UUID 重命名存储，不覆盖原文件），在文件列表中显示原始文件名，上传时间区分

**支持的文件格式：**

| 类型 | 扩展名 |
|------|--------|
| PDF | .pdf |
| Word | .doc, .docx |
| Excel | .xls, .xlsx |
| PPT | .ppt, .pptx |
| 图片 | .jpg, .jpeg, .png, .gif, .webp |
| 纯文本 | .txt |
| Markdown | .md |
| 压缩包 | .zip |

**优先级：** P0

---

#### US-F002: 查看文件列表
**作为**知识库所有者，**我想要**查看已上传的文件列表，**以便**了解知识库中的文件资产。

**验收标准：**
1. **Given** 用户进入文件管理页 **When** 页面加载完成 **Then** 以列表形式显示所有已上传文件，每项展示：文件名、文件类型图标、文件大小（人类可读格式，如 1.2 MB）、上传时间、文件描述（可选）、标签
2. **Given** 文件列表 **When** 文件数量超过 20 条 **Then** 分页显示，每页 20 条，默认按上传时间降序排列
3. **Given** 文件列表 **When** 无已上传文件 **Then** 显示空状态提示"暂无文件，点击上传添加"
4. **Given** 文件列表 **When** 用户按文件类型筛选（全部/PDF/文档/图片/文本/压缩包/其他）**Then** 列表仅显示对应类型的文件

**优先级：** P0

---

#### US-F003: 下载文件
**作为**知识库所有者，**我想要**下载文件，**以便**在本地使用文件内容。

**验收标准：**
1. **Given** 用户在文件列表或文件详情页 **When** 点击"下载" **Then** 浏览器启动文件下载，文件名为上传时的原始文件名
2. **Given** 文件已被删除 **When** 用户尝试下载 **Then** 显示"文件不存在"错误提示

**优先级：** P0

---

#### US-F004: 删除文件
**作为**知识库所有者，**我想要**删除不需要的文件，**以便**释放存储空间并保持知识库整洁。

**验收标准：**
1. **Given** 用户在文件列表 **When** 点击"删除" **Then** 弹出确认对话框，提示"确认删除文件《文件名》？此操作不可撤销"
2. **Given** 确认对话框显示 **When** 点击"确认删除" **Then** 文件被删除（数据库记录 + 磁盘文件），列表刷新，显示成功提示
3. **Given** 确认对话框显示 **When** 点击"取消" **Then** 文件不被删除，对话框关闭

**优先级：** P0

---

### P1 — Should Have

#### US-F005: 文件打标签
**作为**知识库所有者，**我想要**给文件打标签，**以便**与笔记共享标签体系，统一筛选管理。

**验收标准：**
1. **Given** 用户上传文件时或在文件列表操作时 **When** 选择已有标签 **Then** 标签关联到该文件，一个文件可关联多个标签（上限 20 个）
2. **Given** 用户在文件管理页 **When** 按标签筛选 **Then** 文件列表仅显示包含该标签的文件

**优先级：** P1

---

#### US-F006: 按文件名搜索
**作为**知识库所有者，**我想要**通过文件名关键词搜索文件，**以便**快速定位目标文件。

**验收标准：**
1. **Given** 用户在文件管理页的搜索框输入关键词 **When** 提交搜索 **Then** 文件列表仅显示文件名包含该关键词的文件（LIKE 匹配，不区分大小写）
2. **Given** 搜索结果 **When** 无匹配文件 **Then** 显示"未找到相关文件"
3. **Given** 搜索结果 **When** 用户清空搜索框 **Then** 恢复显示全部文件列表

**优先级：** P1

## 4. 非功能需求

### 性能
| 指标 | 目标值 |
|------|--------|
| 文件列表 API 响应（P95） | < 500ms |
| 文件上传（≤ 10MB，P95） | < 3 秒 |
| 文件上传（≤ 100MB，P95） | < 30 秒 |
| 文件下载响应延迟（首字节 TTFB，P95） | < 1 秒 |
| 支持文件总数 | >= 1,000 个 |
| 单用户累计存储空间 | <= 10GB（超出后上传接口返回 507，提示磁盘空间不足） |

### 安全
- 文件存储在服务器本地文件系统（uploads/ 目录），不对外直接暴露静态访问路径
- 文件下载通过 Spring Boot 接口（流式传输），后端控制访问
- 上传文件进行 MIME 类型校验（结合扩展名 + Content-Type 双重校验），防止恶意文件伪装
- 文件名进行安全处理，防止路径穿越攻击（Path Traversal）：存储时使用 UUID 重命名
- 单次上传文件大小限制：100MB（Spring Boot multipart 配置）
- 不执行或解析上传文件内容（无服务端渲染/执行）

### 存储
- 文件存储路径：`{uploads-dir}/{yyyy}/{MM}/{uuid}.{ext}`（按年月分目录，UUID 命名防止冲突）
- 数据库仅存储文件元数据（原始文件名、存储路径、MIME 类型、大小、描述、标签关联、上传时间）
- 不支持文件版本管理（各次上传独立存储，历史不覆盖）

### 可用性
- 与现有系统一致，本地部署
- 文件数据备份由用户自行管理 uploads/ 目录

## 5. 边界与排除项

### 本期包含
- 文件上传：支持 PDF/Word/Excel/PPT/图片/TXT/MD/ZIP，单文件 ≤ 100MB（P0）
- 文件列表：分页、文件类型筛选（P0）
- 文件下载（P0）
- 文件删除：元数据 + 磁盘文件同步删除（P0）
- 文件打标签：复用现有 Tag 体系（P1）
- 按文件名搜索（P1）

### 本期不做
- 文件内容预览（PDF 渲染、图片全屏等）
- 文件重命名
- 文件版本管理
- 文件共享/公开链接
- 文件内容全文提取与搜索（如 PDF 文本索引）
- 批量上传、批量删除
- 文件夹/目录层级管理
- 分类体系与文件关联（仅支持标签关联）
- 笔记与文件混合搜索/混合列表

## 6. 术语表

| 术语 | 定义 |
|------|------|
| 文件（KnowledgeFile） | 系统新增知识实体，包含原始文件名、UUID 存储名、存储路径、MIME 类型、文件大小、描述、关联标签、上传时间 |
| 文件类型（FileCategory） | 基于扩展名归类的文件大类：PDF、文档（Word/Excel/PPT）、图片（JPG/PNG/GIF/WEBP）、文本（TXT/MD）、压缩包（ZIP）、其他 |
| 原始文件名（Original Filename） | 用户上传时的文件名称，用于下载时还原文件名；与存储文件名（UUID）区分 |
| 存储路径（Storage Path） | 文件在服务器文件系统中的实际存储路径，格式：`{year}/{month}/{uuid}.{ext}` |
| 文件描述（Description） | 用户为文件附加的自由文本说明，最多 500 字符，可选 |

## 7. 前置假设

- 现有标签（Tag）体系不修改，文件与标签共享同一张 tag 表；新增 file_tag 关联表
- 服务器本地磁盘有足够空间存储用户上传文件（由部署人员保障）
- 不考虑多用户并发上传场景（单用户系统）
- 文件上传使用 multipart/form-data，通过现有 Spring Boot 服务处理
- uploads/ 目录在部署文档中说明，用户自行备份

## 8. 与现有需求的关系

| 现有 US | 关系 | 说明 |
|---------|------|------|
| US-007 标签管理 | 复用 | 文件可使用现有标签，标签 CRUD 不变 |
| US-008 标签筛选 | 扩展 | 在文件管理页新增按标签筛选文件（与笔记列表的标签筛选独立） |
| US-006 全文搜索 | 不影响 | 全文搜索仅搜索笔记内容，不搜索文件内容 |
| US-004 笔记列表 | 不影响 | 笔记列表与文件列表独立页面，各自维护 |
